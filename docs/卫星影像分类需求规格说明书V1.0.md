## 1. 引言

### 1.1 编写目的

本文档定义"基于深度学习的卫星影像分类系统（EuroSAT_Classification）"的完整功能需求、非功能需求及验收标准，作为以下工作的统一输入：

1. 概要设计与详细设计的依据
2. 开发实现的规格基准
3. 测试用例编写的需求来源
4. 项目验收的判定标准

### 1.2 项目背景

EuroSAT 数据集包含 27,000 张 Sentinel-2 卫星遥感影像，覆盖 10 类土地覆盖类型（年度作物、森林、草本植被、高速公路、工业区、牧场、永久作物、住宅区、河流、海湖）。本项目利用深度学习方法对这些卫星影像进行自动场景分类，判断每张图像所属的土地覆盖类别。

**业务场景**：国土资源调查、土地利用监测、城市规划与环境评估。

**核心任务**：在 GTX 1050 Ti（4 GB 显存）硬件约束下，基于 ResNet18 迁移学习，构建一个高精度、轻量化的卫星影像分类系统，实现从数据加载、模型训练、评估到推理预测的完整流程。

**项目交付范围（一句话）**：在 GTX 1050 Ti 上完成 ResNet18 从 EuroSAT 数据加载、预处理、模型训练、评估到推理输出的完整分类流程，并提供训练监控与可视化能力。

### 1.3 适用范围

- **目标用户**：遥感/GIS 领域算法工程师、遥感影像分析人员
- **适用硬件**：NVIDIA GTX 1050 Ti 4 GB 及以上 GPU，支持纯 CPU 训练与推理
- **适用数据**：EuroSAT RGB 版本（64×64 像素 Sentinel-2 卫星影像），10 类土地覆盖场景
- **操作系统**：Windows 10/11 64-bit（开发主环境），兼容 Linux

## 2. 项目概述

### 2.1 项目名称

**EuroSAT_Classification** — 基于 ResNet18 迁移学习的 Sentinel-2 卫星影像土地覆盖分类系统（GTX 1050 Ti 适配版）

### 2.2 项目目标

1. 在 GTX 1050 Ti（4 GB 显存）上完成 EuroSAT 数据集（27,000 张）的完整训练
2. 训练显存占用 ≤ 3.0 GB（batch_size=256 估算上限），为办公软件预留充足显存
3. 测试集 Top-1 Accuracy ≥ 90%（EuroSAT ResNet18 迁移学习公开参考水平）
4. 支持单张卫星影像快速推理，预测耗时 ≤ 50 ms（含预处理）
5. 提供完整的训练监控（TensorBoard）和评估可视化（混淆矩阵、训练曲线）

### 2.3 数据集说明

| 属性 | 说明 |
|------|------|
| 名称 | EuroSAT (RGB version) |
| 来源 | Sentinel-2 卫星影像 |
| 图像尺寸 | 64×64 像素 |
| 图像格式 | JPEG/PNG（RGB 3 通道） |
| 地面采样距离 | 10 m/像素 |
| 类别数量 | 10 类 |
| 类别名称 | AnnualCrop（年度作物）、Forest（森林）、HerbaceousVegetation（草本植被）、Highway（高速公路）、Industrial（工业区）、Pasture（牧场）、PermanentCrop（永久作物）、Residential（住宅区）、River（河流）、SeaLake（海湖） |

在 GTX 1050 Ti 4 GB 的约束下，以下模型位于 **精度-效率帕累托前沿**（即不存在另一个模型在精度和效率上都优于它）：

```

    │                    ★ ResNet34 (~95.5%)

    │

    │    ★ MobileNetV2 (~93.5%)
94% │ ★ ResNet18 (~93.0%)   ← 当前选择
    │
93% └──────────────────────────────────────────→ 参数量 (M)
    0    5    10    15    20    25    30
```

#### 2.4.4 最终选型决策

**主选模型：ResNet18**（参数量 11.7M，FLOPs 1.8G，EuroSAT 基准约 93~95%）

**选型理由**：
1. **经典可靠**：ResNet 系列是最广泛验证的 CNN 架构，文档丰富、社区成熟、调试便捷。EuroSAT 原始论文即以 ResNet 系列作为主要基准
2. **硬件友好**：11.7M 参数 + batch=256 训练显存 < 1.5 GB，为办公软件预留 > 2.5 GB，远低于 3.0 GB 硬上限
3. **训练效率高**：迁移学习仅训练分类头（~5K 参数），50 epochs 可在约 50 分钟内完成；30 epochs 约 30 分钟即可完成快速验证
4. **精度充足**：93~95% 的 Top-1 Accuracy 已满足土地覆盖分类的工程应用需求（≥ 90%）
5. **升级路径清晰**：冻结骨干→解冻 layer4→解冻全部，三步渐进微调策略成熟

**推荐替代方案（Should 优先级）**：

| 模型 | 参数量 | 预期精度 | 推荐理由 |
|------|--------|---------|---------|
| **MobileNetV3-Large** | 5.4M | ~98.7% | **最优先推荐的替代方案**——参数量仅为 ResNet18 的 46%，精度却高约 5 个百分点，在 EuroSAT 上表现惊人。如果精度是首要目标且希望模型更轻量，应优先选择此模型 |
| **DenseNet-121** | 8.0M | ~97.5% | 密集连接带来极好的特征复用，在 64×64 小图上尤其有效，参数量适中 |

**升级方案（Could 优先级）**：

| 模型 | 参数量 | 预期精度 | 说明 |
|------|--------|---------|------|
| ConvNeXt-Tiny | 28.6M | ~98.7% | 现代 CNN 架构代表，精度与 MobileNetV3-Large 相当，但参数量大 5 倍，适合精度极致追求场景 |
| ResNet50 | 25.6M | ~98.8% | ResNet 家族中坚，精度显著高于 ResNet18，但需适当降低 batch_size 以控制显存 |
| EfficientNetV2-S | 21.5M | ~98.9% | 精度接近 SOTA，但训练配置较复杂（需精细学习率和增强策略），建议有调参经验后启用 |

**否决方案及理由**：

| 模型 | 否决理由 |
|------|---------|
| ViT / Swin Transformer | 参数量大（80M+），需要大规模预训练和大 batch_size，在 4 GB 显存和 EuroSAT（仅 27K 图像）上优势无法发挥 |
| ResNeXt-50 / DenseNet-161 | 精度虽高（~99%），但参数量接近 30M，batch=256 时显存接近上限，性价比较 MobileNetV3-Large 低 |
| 从头训练（无预训练权重） | EuroSAT 的 27K 样本不足以从头训练 ResNet18 达到 90%+ 精度（参考：从头训练 ResNet50 仅约 82%），迁移学习收益约 10 个百分点 |
| 全波段 13 通道模型 | 本期聚焦 RGB 版本快速验证，13 通道需修改输入层和预处理流程，训练数据量和显存需求均翻倍 |

**预训练权重**：ImageNet-1K (IMAGENET1K_V1)，由 TorchVision 自动下载。

**训练策略建议（基于选型结论）**：
- **ResNet18（主选）**：冻结骨干 → 仅训练分类头（~5K 参数），50 epochs，batch=256，FP32
- **MobileNetV3-Large（推荐替代）**：同样冻结骨干 → 训练分类头，因参数量更小可适当增大 batch_size 或减少 epochs
- **升级验证路径**：若 ResNet18 50 epochs 后精度 < 90%，建议依次尝试：① 解冻 layer4 微调 20 epochs → ② 切换为 MobileNetV3-Large → ③ 启用更强的数据增强（CutMix/MixUp）

### 2.5 技术约束

| 约束项 | 具体指标 |
|--------|---------|
| GPU | NVIDIA GeForce GTX 1050 Ti，4 GB GDDR5，CUDA 计算能力 6.1 |
| 训练峰值显存 | ≤ 3.0 GB（batch_size=256, 224×224, FP32） |
| 操作系统 | Windows 10 64-bit（主环境）；Linux 兼容 |
| Python | 3.8.20（严格锁定） |
| 深度学习框架 | PyTorch 1.12.1 + CUDA 11.3 |
| 预训练模型源 | TorchVision (torchvision.models.resnet18) |
| 模型参数量 | ≤ 12M（ResNet18 约 11.7M） |
| 输入策略 | 64×64 双线性上采样至 224×224（匹配 ResNet18 预训练输入） |
| 训练策略 | 迁移学习（冻结骨干、仅训练分类头），FP32 精度（无需 AMP） |
| 批大小 | 256（训练），256（验证/测试） |
| 优化器 | AdamW，初始学习率 1e-4 |
| 损失函数 | CrossEntropyLoss |
| 数据增强 | 随机水平翻转、随机旋转（±15°）、亮度对比度调整 |
| 推理策略 | 单张 64×64 上采样至 224×224 → 模型前向 → softmax → 预测类别 |

> **关于 AMP 的说明**：EuroSAT 影像上采样后为 224×224，batch_size=256 时总显存占用约 1.5~2.0 GB，远低于 4 GB 上限。FP32 精度已足够高效，因此不启用 AMP。AMP 虽可进一步节省约 30% 显存，但会引入额外的 FP16/FP32 转换开销，在当前小图场景下性价比不高。

### 2.6 系统范围

本系统采用模块化设计，遵循高内聚、低耦合原则，划分为 **8 个核心模块**：

| 模块 | 对应文件 | 功能说明 |
|------|---------|---------|

| 评估模块 | `evaluate.py` | 测试集最终评估；Accuracy/Precision/Recall/F1/混淆矩阵计算；分类报告生成（含各类别指标和 Top-3 易混淆类别分析）；误分类样本 CSV 导出 |

| CLI 入口 | `main.py` | 统一命令行接口：`--mode train/evaluate/predict/check` 四种模式路由；`--help` 完整参数说明；环境检测报告生成 |

## 6. 边界情况与异常处理

| 编号 | 场景描述 | 潜在风险 | 处理方案 | 状态 |
|------|---------|---------|---------|------|
| BC-1 | 数据集路径不存在或为空 | 程序无法启动 | 启动前校验数据路径，不存在或 10 个类别文件夹任一为空则终止并给出明确错误信息（预期路径格式） | ⚪ |
| BC-2 | 图像文件损坏或格式不支持 | DataLoader 抛出异常 | 加载时捕获 PIL/PyTorch 图像读取异常，跳过损坏文件并记录文件名到 `skipped_files.txt`；若跳过数量 > 总数 1%，发出 WARNING | ⚪ |
| BC-3 | 某个类别文件夹缺失 | 数据集不完整，无法正确映射标签 | 启动时检查 10 个类别文件夹名称，缺失或多余均报错并列出期望清单 vs 实际清单 | ⚪ |
| BC-4 | 训练中 GPU 被其他程序抢占 | 显存不足导致 CUDA OOM | 训练启动时独占 GPU 0；若仍发生 OOM，自动将 batch_size 减半（最小 32）并重试当前 epoch，记录 WARNING 日志 | ⚪ |
| BC-5 | 预训练权重下载失败（网络不可用） | 无法加载 ImageNet 权重 | 自动回退为 Kaiming 随机初始化，记录 WARNING 日志并说明影响（需要更多 epoch 收敛，预计精度下降 5~10%）；设置 `weights=None` 继续训练 | ⚪ |
| BC-6 | 训练被 Ctrl+C 中断 | 训练进度丢失 | 捕获 KeyboardInterrupt，保存 `emergency_epoch_XXX.pth`（含 model state + optimizer state + epoch）后优雅退出 | ⚪ |
| BC-7 | 验证集/测试集图像尺寸不是 64×64 | 上采样后尺寸不一致 | 上采样逻辑支持任意输入尺寸 → 224×224（非 64×64 的图像也能正常处理），但记录 INFO 日志提示尺寸异常 | ⚪ |
| BC-8 | 推理输入为灰度图（1 通道） | 模型期望 3 通道输入 | 自动检测，使用 `np.stack([img]*3, axis=-1)` 复制为 3 通道，记录 INFO 日志 | ⚪ |
| BC-9 | 推理输入为 RGBA（4 通道） | 模型期望 3 通道输入 | 自动丢弃 Alpha 通道，仅保留 RGB，记录 INFO 日志 | ⚪ |
| BC-10 | 推理结果所有类别置信度都很低（如 max < 0.3） | 可能为域外图像（不属于 10 类中任何一类） | 正常输出结果，但在日志中标注"低置信度预测"及 max confidence 值；设置可配置的置信度告警阈值（默认 0.3） | ⚪ |
| BC-11 | 磁盘空间不足 | 无法保存检查点和日志 | 训练启动前检查输出目录磁盘空间，< 200 MB 拒绝启动；200 MB~1 GB 给出 WARNING 但允许继续 | ⚪ |
| BC-12 | 输出目录包含上次训练残留 | 同名文件被覆盖或混淆 | 每次训练自动创建带时间戳的子目录（`runs/train_YYYYMMDD_HHMMSS/`），避免覆盖；批量推理默认覆盖同名输出文件，但支持 `--no-overwrite` 参数 | ⚪ |
| BC-13 | DataLoader num_workers 过大导致 CPU 内存不足 | 系统卡顿或 OOM Killer | 启动时检测 CPU 核心数，默认 num_workers=min(4, cpu_count/2)；运行中若 CPU 内存超 80%，自动降级 num_workers | ⚪ |
| BC-14 | 验证集某类样本数为 0（分层划分失败） | 该类 Precision/Recall 无法计算 | 分层采样确保每类至少 1 个样本在验证集；若某类总数 < 3 则发出 WARNING 并合并至相邻类别 | ⚪ |

## 附录A 术语表

| 术语 | 定义 |
|------|------|
| EuroSAT | European Satellite Dataset，基于 Sentinel-2 卫星影像的土地覆盖分类数据集，包含 27,000 张 64×64 像素 RGB 图像，共 10 类 |
| ResNet18 | 18 层残差卷积神经网络，通过跳跃连接（skip connection）解决深层网络退化问题，ImageNet Top-1 约 69.76% |
| ResNet | Residual Network，残差网络 |
| Sentinel-2 | 欧洲航天局（ESA）哥白尼计划的多光谱成像卫星，提供 13 个光谱波段的地球观测数据 |
| GSD | Ground Sampling Distance，地面采样距离，表示影像中一个像素对应地面的实际距离 |
| 迁移学习 (Transfer Learning) | 将在大规模数据集上预训练的模型权重迁移到目标任务，通常只需微调分类头即可适应新任务 |
| Fine-tuning | 微调，在预训练模型基础上使用目标数据集继续训练（可冻结部分层） |
| 分类头 (Classification Head) | 模型最后的全连接层，将特征向量映射为类别预测分数 |
| 骨干网络 (Backbone) | 模型的卷积特征提取部分（如 ResNet18 除 fc 外的所有层） |
| 双线性插值 (Bilinear Interpolation) | 一种图像缩放算法，通过周围 4 个像素的加权平均计算新像素值，比最近邻插值更平滑 |
| CrossEntropyLoss | 交叉熵损失函数，多分类任务的标准损失，结合 LogSoftmax + NLLLoss |
| Top-K Accuracy | 预测概率最高的前 K 个类别中包含真实标签的比例。Top-1 即常规准确率 |
| 混淆矩阵 (Confusion Matrix) | N×N 矩阵，行表示真实类别，列表示预测类别，(i,j) 表示真实为 i 类但被预测为 j 类的样本比例 |
| AdamW | Adam 优化器的改进版本，将权重衰减与自适应学习率解耦，提供更好的泛化性能 |
| ReduceLROnPlateau | 学习率调度策略：当监控指标停止改善时自动降低学习率 |
| Early Stopping | 早停策略：验证集指标连续 N 轮不提升时自动停止训练，防止过拟合 |
| MoSCoW | Must/Should/Could/Won't 优先级分级方法，用于需求优先级管理 |
| Top-1 Accuracy | 模型预测概率最高的类别与真实标签一致的比例 |
