# 技术架构说明

## 一、四层架构

```
┌─────────────────────────────────────────────────────────────┐
│                    表 示 层 (Presentation)                    │
│  main.py                                                      │
│  职责：CLI 参数解析 → 模式路由 → 调用应用层 → 结果输出       │
│  模式：train | evaluate | predict | check                     │
├─────────────────────────────────────────────────────────────┤
│                    应 用 层 (Application)                     │
│  train.py              evaluate.py          predict.py        │
│  职责：训练编排          职责：评估编排        职责：推理编排  │
├─────────────────────────────────────────────────────────────┤
│                    领 域 层 (Domain)                           │
│  model.py              data.py              utils.py          │
│  职责：模型构建          职责：数据全流程      职责：通用工具  │
├─────────────────────────────────────────────────────────────┤
│               基 础 设 施 层 (Infrastructure)                  │
│  config.py                                                    │
│  职责：YAML 配置加载/校验 → 不可变 Config 对象                │
└─────────────────────────────────────────────────────────────┘
```

**依赖规则**：上层可依赖下层；下层不可依赖上层；领域层仅依赖基础设施层；同层通过应用层协调，不直接相互调用。

## 二、模块依赖关系

```
                    ┌──────────────┐
                    │   main.py    │
                    │  (CLI 入口)   │
                    └──┬───┬───┬──┘
                       │   │   │
              ┌────────▼┐ ┌▼───────┐ ┌▼──────────┐
              │ train   │ │evaluate│ │ predict    │
              │ .py     │ │.py     │ │ .py        │
              └──┬──┬───┘ └┬──┬───┘ └┬────┬─────┘
                 │  │      │  │      │    │
        ┌────────▼┐ │ ┌────▼──▼──┐ ┌▼────▼──────┐
        │ model   │ │ │  data    │ │  utils     │
        │ .py     │ │ │  .py     │ │  .py       │
        └─────────┘ │ └──────────┘ └────────────┘
                    │
              ┌─────▼──────┐
              │  config    │
              │  .py       │
              └────────────┘
```

**循环依赖检查**：0 个。所有箭头单向。data/model/utils 仅依赖 config，不依赖任何业务模块。

## 三、数据流

### 训练数据流

```
config.yaml ──→ config.py ──→ Config 对象
                                   │
EuroSAT/类别文件夹 ──→ data.py ──→ DataLoader ──→ train.py ──→ model.py
  (10个子目录)            │           (batch=256)         │
                          ▼                               ▼
                    ImageFolder                     checkpoints/
                    7:1:2 分层划分                   best_model.pth
                    64→224 BILINEAR                  logs/
                    ImageNet 标准化                  runs/
                    数据增强(仅训练)                 TensorBoard
```

### 推理数据流

```
影像 .jpg/.png ──→ predict.py ──→ 预处理 ──→ model 推理
   (64×64)            │           (224×224       (softmax)
                      │           标准化)            │
                      │                             ▼
                      │                        Top-K 类别
                      │                        + 置信度
                      │                             │
                      └──────────┬──────────────────┘
                                 ▼
                          CSV / 控制台输出
```

## 五、关键设计决策

| 决策 | 理由 |
|------|------|
| 8 模块扁平结构 | 每模块单一职责，<400 行，无包目录开销 |
| frozen dataclass 配置 | 运行时不可变，IDE 字段补全，防拼写错误 |
| 不启用 AMP (混合精度) | Pascal 架构无 FP16 Tensor Core，无加速收益 |
| 仅用 torchvision.transforms | 分类任务增强够用，不引入 albumentations |
| BILINEAR 上采样 | 自然平滑，与 ImageNet 预训练权重兼容 |
| os.replace 原子写入 | Windows 同磁盘原子操作，防 checkpoint 损坏 |
| 8→4 渐进微调策略 | PEFT 先行 → layer4 微调 → 全量微调，风险可控 |
